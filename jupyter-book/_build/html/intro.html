소개¶

컴퓨터 과학과 알고리즘¶

컴퓨터 과학의 기본 핵심은 문제해결을 위한 컴퓨터 활용이다. 그리고 컴퓨터 과학자는 주어진 문제를 해결하는 알고리즘을 연구하고 개발한다. 여기서 알고리즘은 주어진 문제problem의 모든 사례instances를 일괄적으로 해결하는 단계별 명령의 목록을 의미한다. 예를 들어 제곱근을 계산하는 문제는 모든 사례, 즉 임의의 수에 대한 제곱근 계산을 수행하는 알고리즘에 의해 해결된다.

문제를 해결하는 알고리즘이 존재하지만 해당 알고리즘을 실행할 때 적절한 시간 내에 실행이 멈추지 않거나 메모리를 너무 많이 사용해서 프로그램 실행이 멈추는 경우도 발생한다. 그런 알고리즘을 구별해내는 일이 해결책을 찾아 내는 일만큼 중요하다. 즉, 알고리즘을 작성하는 능력 뿐만 아니라 다양한 알고리즘을 분석, 비교하여 가장 좋은 알고리즘을 선택하는 능력 또한 매우 중요하다. 이를 위해 다양한 문제를 해결하는 알고리즘을 배우고 주어진 알고리즘을 분석하는 기법을 학습해야 한다.

반면에 (여기서 자세히 다루지 않겠지만) 컴퓨터로 해결이 불가능한 문제도 존재한다. 따라서 컴퓨터 과학은 문제의 해결책뿐만 아니라, 해결책이 존재하지 않는 문제에 대한 연구도 포함한다. 해결이 가능한 문제를 계산가능한computable 문제라고 말한다. 그리고 특정 컴퓨터나 특정 프로그래밍언어에 의존하는 해결책이 아니라 어느 정도의 기본만 갖추고 있는 컴퓨터나 프로그래밍언어를 이용하면 해결할 수 있는 문제를 의미함에 주의해야 한다.

알고리즘과 자료구조¶

주어진 문제를 해결하는 알고리즘을 특정 프로그래밍언어로 구현하려면 다음 두 가지가 요구된다.

  • 문제의 사례들을 표현하는 데에 필요한 자료구조data structure

  • 원하는 결과를 얻는데 필요한 단계step를 묘사하는 명령문

대부분의 프로그래밍언어는 아래 데이터들에 대한 자료구조와 함께 관련 연산자를 제공한다. 예를 들어 사칙연산자와 논리 연산자 등이 기본으로 포함된다.

  • 정수

  • 문자열

  • 부동소수점

  • 부울값

반면에 원하는 결과를 생성하는 단계들의 묘사에 필요한 프로그래밍언어의 기본 명령문은 다음과 같다.

  • 변수 할당

  • if...else... 등의 조건 선택 처리

  • for, while 등의 명령문 반복 처리

원칙적으로는 언급된 기본 자료구조와 명령문만을 이용하여 모든 문제를 해결할 수 있다. 하지만 복잡한 문제를 효율적인 알고리즘으로 해결하려면 코드 추상화와 보다 복잡한 자료구조가 요구된다.

추상화¶

프로그래밍에서는 많은 종류의 추상화를 사용하지만 여기서는 절차 추상화와 데이터 추상화 두 종류만 언급한다.

절차 추상화

주어진 (해결가능한) 문제는 적절한 API(Application Programming Interface)를 조합하여 해결한다. 반면에 각각의 API를 어떻게 구현하는가는 여기서는 관심대상이 아니다.

예를 들어, 제곱근을 계산하는 함수 sqrt()는 math 모듈에 정의되어 있으며 아래와 같이 활용한다.

import math

math.sqrt(16)

4.0

위 코드는 절차 추상화procedural abstraction를 보여준다. 즉, sqrt() 함수가 어떻게 정의되었는가는 모른 채 제곱근을 계산하는 함수라는 사실과 어떻게 활용하는가를 알기만 하면 된다.

데이터 추상화

추상 자료형Abstract data type(ADT)은 특정 데이터 및 그 데이터와 관련된 연산자와 함수들을 하나로 묶어 다루는 방법을 묘사하는 객체(대상)다. 그리고 추상 자료형을 이용하여 적절한 데이터를 정의하고 다루는 기법을 데이터 추상화data abstraction라고 한다.

사용자의 입장에서 데이터, 연산자, 함수가 어떻게 구현되었는가는 중요하지 않다. 대신에 해당 데이터가 무엇을 표현하며 관련 연산자와 함수의 기능이 무엇인가만 알면 된다. 이런 의미에서 데이터 추상화를 데이터 캡슐화data encapsulation라고 부르기도 한다.

아래 그림에서 볼 수 있듯이 사용자는 추상 자료형이 제공하는 연산자나 함수를 이용하여 추상 자료형의 인터페이스와 교감하는 방식으로 데이터를 활용할 뿐이다. 추상 자료형이 구체적으로 어떻게 구현되었는가는 사용자 입장에서는 일반적으로 중요하지 않다.

자료구조¶

자료구조는 추상 자료형을 직접 하나의 클래스로 구현한 것을 가리킨다. 추상 자료형을 하나의 클래스로 구현하는 방식이 다양할 수 있다. 하지만 앞서 설명하였듯이 구현된 자료구조는 기본적으로 구현 방식과 무관하게 사용하고자 하는 데이터와 관련 함수의 기능을 적절하게 제공만 하면 된다.

추상 자료형과 자료구조를 분리해서 다루는 이유는 데이터 추상화를 이용하여 다루고자 하는 데이터의 기본 속성과 사용법을 보다 구현과 관련된 번잡한 내용 없이 보다 일반적인(추상적인) 방식으로 설명할 수 있도록 하기 위함이다.

물론 주어진 문제를 해결하는 알고리즘을 구현하려면 적절한 추상 자료형을 구체적으로 구현한 클래스를 작성해야 한다. 하지만 구현된 클래스의 활용법은 해당 추상 자료형의 기본 속성과 함수들의 기능을 이해하기만 하면 된다.

파이썬 프로그래밍언어¶

언급되는 모든 알고리즘과 자료구조의 구현은 파이썬 프로그래밍언어를 사용한다. 파이썬은 객체지향 프로그래밍(OOP) 언어이며 중요한 기본 자료형(자료구조)들을 제공한다. 또한 객체지향 언어로써 파이썬은 클래스class를 이용하여 자료구조를 정의하는 방식을 지원한다.

많은 객체지향 프로그래밍언어들 중에서 파이썬을 사용하는 이유는 파이썬이 알고리즘을 가장 자연스럽게 표현해주는 언어이기 때문이다.
